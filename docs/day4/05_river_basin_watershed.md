# River Basins and Stream Ordering

River basins are organized hierarchically: large continental basins are composed of multiple nested sub-basins, which are in turn divided into watersheds and minor catchments. To analyze these drainage networks, stream ordering systems are used to classify channel segments based on their branch complexity. This section covers basin delineation workflows and the Strahler and Shreve ordering systems.

---

## 1. Delineation Workflows in GRASS and SAGA

In QGIS, watershed boundaries are typically delineated using GRASS or SAGA algorithms within the Processing Toolbox. The standard workflow includes:

### GRASS `r.watershed`

A highly robust, multi-functional tool that calculates flow direction, flow accumulation, stream segments, and sub-basin boundaries simultaneously.

* **Threshold Input:** You define the minimum size of an exterior drainage basin (in cell counts) to control stream density.

* **MFD Option:** Supports calculating flow direction using both D8 and Multiple Flow Direction (MFD) models.

### GRASS `r.water.outlet`

Delineates a single catchment boundary upstream of a specific pour point coordinate:

1. Locate the coordinate coordinates of your outlet (e.g., a river gauge position).

2. Input the D8 flow direction raster generated by `r.watershed`.

3. Click **Run**. The output is a binary raster polygon isolating the contributing basin area.

### SAGA `Upslope Area`

Similar to `r.water.outlet`, SAGA's **Upslope Area** calculates the contributing area draining to a single target coordinate cell. It supports multiple flow routing options (D8, FD8, D-Infinity).

## 2. Strahler Stream Ordering

Developed by Arthur Strahler in 1952, this is the most widely used hierarchical system for classifying stream networks.

```text
    STRAHLER ORDERING RULES
    
    (1) ---\
            (2) ---\
    (1) ---/        \
                    (3) ---> Outflow (Order 3)
    (2) ------------/
    
    (1) ----(3) --------> Outflow (No Change, stays Order 3)
```

### The Strahler Rules:

1. **Order 1:** Headwater streams without any upstream tributaries.

2. **Order N+1:** Formed only when two streams of the identical order ($N$) intersect. (e.g., when two Order 2 streams merge, the downstream channel becomes Order 3).

3. **No Change (Order N):** When a lower-order stream ($M$) merges with a higher-order stream ($N$, where $M < N$), the downstream channel remains at the higher order ($N$). (e.g., an Order 1 stream merging into an Order 2 channel does not increase the order; it stays Order 2).

* **Hydrological Application:** Strahler order helps classify stream size. Headwaters (Orders 1-2) are steep and flash-flood prone; mid-order channels (Orders 3-5) exhibit lower slopes; high-order rivers (Order 6+) represent major sluggish valley floor channels.

## 3. Shreve Stream Ordering (Stream Magnitude)

Developed by Ronald Shreve in 1966, this is an additive ordering system that reflects the total number of headwater tributaries draining through a channel.

### The Shreve Rules:

1. **Magnitude 1:** Headwater streams.

2. **Magnitude A + B:** When two streams of magnitudes $A$ and $B$ intersect, the downstream channel is assigned the sum of their magnitudes ($A + B$).

```text
    SHREVE MAGNITUDE RULES
    
    (1) ---\
            (2) ---\
    (1) ---/        \
                    (5) ---> Outflow (Magnitude 5)
    (3) ------------/
```

* **Hydrological Application:** Unlike Strahler order, which only increases when identical branches merge, Shreve magnitude counts *every* contributing upstream headwater stream. Consequently, Shreve magnitude values are highly correlated with cumulative river flow volumes, drainage area size, and total discharge.
